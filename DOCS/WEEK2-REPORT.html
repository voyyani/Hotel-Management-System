Challenges & Solutions

Challenge: Complex RLS Policy Logic

Problem: Needed different access levels for 4 user roles with overlapping permissions.

Solution: Created helper functions (auth.is_admin(), auth.is_manager_or_admin()) to simplify policy conditions. Used SECURITY DEFINER for elevated privileges check.

Time: 2 hours | Resolved ✓

Challenge: Double Booking Prevention

Problem: SQL date range overlap detection with multiple edge cases.

Solution: Used PostgreSQL OVERLAPS operator in check_room_availability() function. Created BEFORE INSERT/UPDATE trigger to enforce at database level.

Time: 1.5 hours | Resolved ✓

Challenge: Realtime Publication Configuration

Problem: Realtime not triggering after initial setup.

Solution: Used ALTER PUBLICATION supabase_realtime ADD TABLE syntax. Verified with pg_publication_tables query. Ensured RLS policies don't block Realtime events.

Time: 1 hour | Resolved ✓

Challenge: Full-Text Guest Search

Problem: LIKE queries too slow for guest name search.

Solution: Implemented GIN index with pg_trgm extension for trigram similarity search. Creates index on concatenated first_name + last_name.

Time: 1 hour | Resolved ✓

Total Time on Challenges: 5.5 hours (~14% of week) | Success Rate: 100% resolved

Learnings & Insights

PostgreSQL Advanced Features

  • • Triggers are powerful for business logic enforcement
  • • JSONB columns provide schema flexibility
  • • Partial indexes drastically reduce index size
  • • SECURITY DEFINER enables safe privileged operations
  • • OVERLAPS operator perfect for date range checks

Row-Level Security Mastery

  • • RLS policies are evaluated in OR fashion
  • • Helper functions simplify complex policies
  • • auth.uid() provides current user context
  • • Cannot bypass RLS with direct table access
  • • RLS applies to Realtime events automatically

Database Design Principles

  • • Normalization prevents data anomalies
  • • Strategic denormalization improves performance
  • • UUID primary keys better than SERIAL for distributed systems
  • • Constraints catch errors at database level
  • • Timestamps on all tables aid debugging

Supabase Platform Insights

  • • Real-time with RLS is game-changing
  • • SQL Editor excellent for rapid prototyping
  • • Type generation speeds up frontend development
  • • Free tier generous for development
  • • Migration workflow smooth with CLI
